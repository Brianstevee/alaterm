HELP for Alaterm - Arch Linux ARM in Termux

Help version 1. February 27, 2020

This help file is intended for those who already installed Alaterm, and are reading it from within the installation. But if you are reading the raw text at GitHub, this information is incomplete.

Topics:
1. Capabilities and Limitations
2. LXDE Desktop, taskbar, and Menu
3. Installing and removing software packages

1. Capabilities and Limitations

The LXDE Desktop works with touchscreens. Usage is tricky. First, you move the cursor by sliding your finger. Then, when the cursor is over its destination, you tap. The trick is that your finger does not have to be over the cursor.

Since most of the useful applications involve images, it is best to use Alaterm on a relatively large-screen device, with external keyboard and mouse. My own device is a 10.1 inch tablet. Right-click is supported!

Alaterm cannot perform any system manipulation unless that privilege is granted to Termux. For example, at the time I write this, Termux does not have the privilege to start most services and daemons, even if the software is installed. Therefore, you will be unable to start a CUPS printer in Alaterm, or a MIDI instrument controller. That may or may not change in the future, but it is beyond the control of Alaterm.

When Alaterm is launched, you can manipulate some programs directly from the command line. In that situation, you do not need to launch VNC Viewer, as there is nothing to see.

With VNC Viewer, you can see the LXDE graphical desktop at port 127.0.0.1:5901. But where is the button for exit or logout? There is none! You leave Alaterm from the terminal, using logout to return to Termux.

Since Android is always active, you can flip back and forth between Alaterm and ordinary Android apps. So, you can check your E-mail in Android, while Alaterm sits idle or does something else. Likewise, you can do something in Alaterm, while an Android app plays music to Bluetooth earbuds. But remember that your device does not have the resources of a desktop computer. You have less RAM, no hard drive, and many background processes. That will limit how much you can do.

As an ordinary user, you are granted superuser privileges within Alaterm. You can create, delete, and edit most Alaterm system files. In rare circumstances, you may use sudo for more privileges. No password needed. In the rarest of circumstances, you may fool Alaterm into thinking that you ar the root, by using su with password root. But this is only a deception. You do not gain root privileges in Android.

Without launching Alaterm, Termux can create, delete, or edit files within the Alaterm installation. So there is no point to having secure passwords. If you uninstall Termux, then Alaterm is also deleted.

2. LXDE Desktop, taskbar, and Menu

At the left corner of the taskbar is the application menu. Use it to launch programs. The initial setup is pre-configured with:
GPicView image viewer
Gnome Font Viewer
Evince document viewer for PDF
Gnome Calculator
Leafpad plain text editor
Geany text editor for programmers
GHex hex editor
NetSurf web browser
PCManFM file manager
Plus LXDE utilities for changing the look and feel of the desktop.

The Menu is pre-configured so that it hides features that are known to fail in Alaterm. Most of them are system-level services, particularly daemons.

Next to the LXDE Menu is the File Manager, named PCManFM. Several useful locations are pre-configured as shortcuts. The home folder for Alaterm is different than the Termux home. Likewise, Alaterm and Termux have their /usr software directories in different locations. There are shortcuts to places where Android allows file sharing.

Because Alaterm sits within Termux, which sits within Android, Alaterm can only do what Termux and Android allow it to do. Thus, PCManFM can only see what Termux and Android allow it to see.

Android does not allow Alaterm to start most system services. This is why audio-visual programs, and CUPS printing, are unlikely to work in Alaterm. If Termux ever gains those privileges, than perhaps Alatarm can take advantage.

Bluetooth and Internet are managed by Android at all times. If an Alaterm program requires Bluetooth or Internet connection, it will probably work.

3. Installing and removing software packages

Arch uses pacman to install, remove, and query software packages. Its syntax can be obscure.

The most commonly-used pacman commands are case-sensitive:
pacman -Syu updates all installed software.
pacman -S packagename installs packagename, or re-installs it if requested.
pacman -R packagename removes packagename, unless other installed packages depend on it.
pacman -Rsc packagename removes packagename, its cache, and any dependencies that were installed at the same time. Use this when you try a package but it does not work as expected.

Search the Internet for useful how-to info, such as here

4. Suggested Software

For a high-end image editor, use GIMP:
pacman -S gimp

You will get a very up-to-date version of GIMP, with full capabilities. Others will envy you. However, your device does not have enough computing power to handle multiple large images or many layers.

The Inkscape vector graphics editor can also be installed. It is a good drawing program, but a bit hard to understand if you have never used it before.
pacman -S inkscape

The Hugin, panorama photo stitcher, can be installed. However, photo stitching is inherently demanding of memory, CPU, and read-write temporary file access. For those reasons, you may find that Hugin is too much for your device.
pacman -S hugin

Using the command-line, ImageMagick can perform a great variety of image editing tasks. ImageMagick, and the similar GraphicsMagick, are widely used when image handling must be automated by script: pacman -S imagemagick

Many modern handheld devices save screenshots and photos in heic/heif format, also known as h265. Alaterm image applications have the ability to read and write this format, in principle. In pracice, there is a bug in the software, at the time I write this. The easiest solution is to let an Android app convert heic/heif to something such as jpg or png, which can be edited in Alaterm. Another possibility is an online image conversion service. But if you really need Alaterm software to read and write heif/heic images, you can compile the necessary software. This takes about 250MB space and 40-60 minutes. Fortunately, there is a built-in script that does all the thinking for you:
compile-h265

You can create sophisticated new fonts, or edit existing fonts, using the most excellent FontForge:
pacman -S fontforge

If you wish to install additional fonts, there are several ways to do it. One possibility is that you can simply install a prepared font package. Or, fonts can be manually installed. The various possibilities are described on this page, but keep in mind that the selection of prepared packages may be smaller in Arch Linux ARM than in the full Arch Linux.

The TeX typesetter, also known as LaTeX and TeXLive, works well. It can process via command line, or with GUI programs. To install TeX:
pacman -S texlive-most

That will ask you which parts you require. TeXLive is inherently a very large installation, perhaps up to a Gigabyte. Do not install parts that you do not expect to need. Although LuaTeX is not mentioned by name, it is part of some other packages there.

Then, you may install a GUI, one or more of:
pacman -S texworks texmaker texstudio

Paths in Arch are different than paths used by some other distributions. You may need to configure the GUI programs so that they can find the TeX installation.

There is no tlmgr program for managing TeX. There is a user-created alternative to tlmgr, but I have found that occasional updates via pacman are adequate.

You can install the LibreOffice suite:
pacman -S libreoffice-still

Do not install libreoffice-fresh. Do not install Java or its runtime.

Before you install the very large LibreOffice suite: Are you sure you need it? Android has many native apps that can create documents, show slide presentations, calculate spreadsheets, and even maintain a small database. Many of them are linked to "cloud" storage via Microsoft or Google, or other providers. In some cases, your little device sends information to the greate computer in the sky, which does the heavy work for you, saves the information, and returns processed results to you. The only reason to use LibreOffice locally is if you are not often connected to cloud storage, or you have existing documents that must use the LibreOffice file formats.

The Menu will show LibreOffice, and LibreOffice Writer. The other LibreOffice programs are there, but not shown on the Menu. Launching the main LibreOffice provides a choice.

As with all other Arch programs, LibreOffice is designed for desktop computers with large resources. Do not even dream of creating a massive relational database on your device, even though the software is there.

You might or might not be able to print to a Bluetooth printer. You can definitely export files to PDF, move the PDF to where Android can see it, then print from there.

If you intend to play or manipulate multimedia files, then you are in for a world of hurt.

At the time I write this, in late February 2020, Alaterm has only partial support for multimedia of any kind. This is because all multimedia must pass through Anrdoid, via Termux. Numerous Android apps have much better multimedia support. So, there is no reason to spend great effort doing multimedia in Alaterm.

At some future time, maybe Termux will have greater multimedia support, which can be brought into Alaterm. But do not expect it, and do not ask for it.
