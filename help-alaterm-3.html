3.1 Using pacman

  • Software is installed, removed, and updated from the terminal command line. The basic command is pacman. Its syntax can be obscure. Here are a few popular commands. They are case-sensitive:
    pacman -Syu updates all installed software.
    pacman -S packagename installs a package, or re-installs it if requested.
    pacman -R packagename removes a package, unless other installed packages depend on it.
    pacman -Rsc packagename removes packagename, its cache, and any dependencies that were installed at the same time. Use this when you try a package but it does not work as expected.

3.2. Installing Fonts

  • Alaterm does not install hundreds of fonts that you do not need. You may install additional TrueType or Open Type fonts.
  • To search for font packages in alaterm:
    pacman -Ss font
    That will return the names of packages that have font in the name or description. Some of them are font-handling software (rather than fonts), and some are specialized or bitmap fonts. For ordinary purposes in word processing and image labeling, you want TrueType or OpenType fonts.
  • Among the available font packages, I recommend the Linux Libertine fonts:
    pacman -S ttf-linux-libertine
  • Otherwise, you can install fonts manually.
    • Terminology: A font family consists of one or more font files, with the same basic name. Thus, "Dribble Dorf Serif" and "Dribble Dorf Italic" and "Dribble Dorf Bold" are all members of the "Dribble Dorf" font family. But "Drabble Dorf Serif" and "Dribble Drat Serif" belong to different families. In many cases, members of a font family are distributed as a single download.
    • Put the font family in a single folder, with the base family name. No spaces in the folder name. So, the folder might be named "dirbble-dorf" (capitalization does not matter).
    • Put that folder into /usr/share/fonts.
    • Then update the font database:
      fc-cache
    • Check that they are now available:
      fc-list
  • If you are using TeXLive, be aware that TeX has its own method for fonts. But you knew that.

3.3. GIMP Image Editor

  • For a high-end image editor, use GIMP:
    pacman -S gimp
  • You will get a very up-to-date version of GIMP, with full capabilities. Others will envy you. However, your device does not have enough computing power to handle multiple large images or many layers.

3.4. Inkscape Vector Drawing

  • The Inkscape vector graphics editor can also be installed. It is a good drawing program, but a bit hard to understand if you have never used it before.
    pacman -S inkscape

3.5. LibreOffice

  • You can install the LibreOffice suite:
    pacman -S libreoffice-still
  • Do not install libreoffice-fresh. Do not install Java or its runtime.
  • Before you install the very large LibreOffice suite: Are you sure you need it? Android has many native apps that can create documents, show slide presentations, calculate spreadsheets, and even maintain a small database. Many of them are linked to "cloud" storage via Microsoft or Google, or other providers. In some cases, your little device sends information to the greate computer in the sky, which does the heavy work for you, saves the information, and returns processed results to you. The only reason to use LibreOffice locally is if you are not often connected to cloud storage, or you have existing documents that must use the LibreOffice file formats.
  • The Menu will show LibreOffice, and LibreOffice Writer. The other LibreOffice programs are there, but not shown on the Menu. Launching the main LibreOffice provides a choice.
  • LibreOffice is designed for desktop computers with large resources. Do not even dream of creating a massive relational database on your device.
  • You might or might not be able to print to a Bluetooth printer. You can definitely export files to PDF, move the PDF to where Android can see it, then print from there.

3.6. TeXLive and GUIs

  • The TeX typesetter, also known as LaTeX and TeXLive, works well. It can process via command line, or with GUI programs.
  • The command-line TeXLive files can be installed directly into Termux, without the need for alaterm. Or they can be installed into alaterm. Or both!
  • If you already have a good, working TeX installation in Termux, and if you intend to use it by command-line and/or TeXWorks GUI, then you do not need to install TeX in alaterm.
    • You can directly access the Termux command-line TeX programs, by typing its commands in alaterm.
    • You can install the TeXWorks GUI in alaterm, and it will be able to use the TeX software from Termux.
  • If you do not already have a good, working TeX installation in Termux, then you can install it in alaterm:
    pacman -S texlive-most
    That will ask you which parts you require. TeXLive is inherently a very large installation, perhaps up to a Gigabyte. Do not install parts that you do not expect to need.
  • You can install TeXMaker and TeXStudio in alaterm, but in these cases you will also have to install TeX into alaterm.
  • There is no tlmgr program for managing TeX in alaterm.
  • The resulting PDF files can be viewed by Evince, called "Document Viewer" in the LXDE Menu.

3.7. Selected Other Programs

  • ImageMagick. This command-line command-line image editor can perform a great variety of tasks. ImageMagick, or the similar GraphicsMagick, are widely used when image handling must be automated by script:
    pacman -S imagemagick
  • Hugin. This is a panorama photo stitcher. However, photo stitching is inherently demanding of memory, CPU, and read-write temporary file access. You may find that Hugin is too much for your device.
    pacman -S hugin
  • FontForge. You can create sophisticated new fonts, or edit existing fonts, using this excellent font editor:
    pacman -S fontforge
  • There are many other programs that can be installed. But understand that you can only install packages built for your device, from the Arch Linux ARM project. Not all Linux programs, indeed not all Arch Linux programs, are available.

3.8. Multimedia: Good and Bad News

  • If you intend to play or manipulate multimedia files, then you are in for a world of hurt. For this reason, alaterm does not come with multimedia support.
  • Since your Android device already has high-quality multimedia support, via native Android apps, there is no reason to use alaterm.
  • The good news: If you do not need to play audio-video, but merely wish to edit the file, that can be done using the command-line program ffmpeg:
    pacman -S ffmpeg
  • You can find many web pages describing how to use ffmpeg, such as this one.

3.9. HEIF/HEIC Images, h265

  • Many modern handheld devices save screenshots and photos in heic/heif format, also known as h265. This is especially true for Apple phones.
  • At the time I write this, the native alaterm software for handling heif/heic is rejected by Android security policies. This is not a fundamental problem, and perhaps it will be fixed in the future, if the Arch Linux ARM project does it.
  • In the meantime, you have choices:
    • 1. If you absolutely must open or save heic/heif still images within alaterm, you can compile the fixed version of the necessary software. The compiler suite takes up about 250MB space, and compilation takes about 25 minutes. At the GitHub alaterm project page, download script compile-libde265. Then:
      bash compile-libde265
      If your existing installation was already fixed, the script will inform you, then exit. Otherwise, it will ask for confirmation, then download and install the compiler suite and the libde265 source code. Everything is automatic. When the script finishes, it runs a quick test, and lets you know the result.
    • 2. If you have Android 9 or later (not sure about 8), then there are numerous Android apps capable of opening heic/heif and exporting or saving to another format such as jpeg. Some of them will convert jpeg to heic/heif.
    • 3. There are online image converter services. Upload the heic/heif, then download the image converted to jpeg or whatever. Also works in reverse.
