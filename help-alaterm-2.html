2.1. Touchscreen vs. Mouse

The LXDE Desktop works with touchscreens. Usage is tricky. First, you move the cursor by sliding your finger. Then, when the cursor is over its destination, you tap. The trick is that your finger does not have to be over the cursor.

Since most of the useful applications involve images, it is best to use Alaterm on a relatively large-screen device, with external keyboard and mouse. My own device is a 10.1 inch tablet. Right-click is supported, with context menus.

2.2 Screen Dimensions and Orientation

The Desktop is initially configured for 1280x800 pixel screens. This assumes that you use your device in landscape mode, like a laptop. Many modern devices have high-resolution screens. My own is 1920x1200. However, use of full resolution will cause everything to appear very tiny, because the software assumes that you are viewing on a large-screen office computer.

If you need a different screen resolution, particularly if your device has an aspect ratio other than 1.6:1, there are two wayts to change it.

From the LXDE Menu, Preferences, Change Default Resolution. This offers you a choice of several values. You can try a value, and if it works for you, save it. The saved value will then be used each time you login.

If none of the choices is what you need, then you can make a custom choice. This requires using the terminal, rather than the Menu. Command:

default-resolution WxH

where W and H are three or four digit numbers, representing the desired width and height. The effect is not immediate. To see it, you have to logout then re-launch alaterm. If you made a bad choice, you can re-try the command, or use the Menu. Remember that the initial default resolution is 1280x800.

The LXDE Desktop does not understand device rotation. If you set the resolution in landcape orientation, then it will not fit properly if you rotate your device to portrait orientation, like a phone. So if you normally intend to use alaterm in portrait orientation, you should choose a default resolution with its height greater than its width.

2.3 LXDE Applications Menu

At the left corner of the taskbar is the applications menu. Use it to launch programs, and to change some preferences. The initial setup is pre-configured with:
GPicView image viewer
Gnome Font Viewer
Evince document viewer for PDF
Gnome Calculator
Leafpad plain text editor
Geany text editor for programmers
GHex hex editor
NetSurf web browser
PCManFM file manager
Preferences, for changing the look and feel of the desktop.
HELP, which invokes this help file.

Installation of additional software is performed from the command terminal, as described below.

The Menu can be edited using PCManFM File Manager. However, this is not recommended, since it requires some expertise.

LXDE installs a few utility programs that only work in office desktop computers. They cannot be uninstalled, since LXDE expects them to be there. For your convenience, Menu items leading to these useless programs have automatically been deleted.

2.4 PCManFM File Manager

Next to the LXDE Menu is the File Manager, named PCManFM. Several useful locations are pre-configured as shortcuts:
Home is your alaterm home folder.
Desktop is your alaterm Desktop, a sub-folder of Home.
Android Shared leads to the Android directory that contains Documents, Downloads, Pictures, and other on-device locations that can be accessed from alaterm.
Removable Media leads to an Android directory containing a removable SD card or USB pendrive.
Termux Home is the home folder for Termux. This is the initial location when Termux launches.
Trash leads to Trash at ~/.local/share/Trash.

Note that only Android can mount media. So if you have a removable media device, be sure that it is mounted and recognized by Android, before you launch alaterm. Also note that Android gives obscure folder names to removable media, and also includes folders leading to inacessible locations. Finding your removable device is a matter of trial and error.

2.5 Trash? Rubbish!

The creator of PCManFM uses the UK term rubbish instead of the US term trash.

There is no animated Trash Bin. This is because the necessary software would violate Android system policies, by attempting to access prohibited directories. Remember that the LXDE Desktop is not specifically written for Android.

The Trash shorcut leads to alaterm Trash. It does not include Android Trash, if it exists. Even though you can send items to Trash easily, restoring items and emptying the Trash can only be done by command terminal. The README-TRASH file describes the commands, It is visible when you enter Trash.
